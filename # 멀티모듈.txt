# 멀티모듈

멀티모듈
특정 기능을 중심으로 독립적으로 운영될수있는 구성요소의 단위 - 모듈

왜 필요할까?
-> 모듈화를 통해 책임과 기능 분리
-> 각 모듈이 최소한의 의존성을 갖도록 구성가능
-> 해당 모듈에 설정된 의존성을 강제적으로 지키며 개발가능


의존성관리의중요성
의존성이 하나의 모듈에 있으면, 불필요한 의존성을 사용할 수 있음
자동구성에 의해 빌드, 실행에 영향을 줄 수 있음


실습
모듈 세가지
API : 사용자의 Http 요청처리, 부트 실행
CORE : 핵심 비즈니스로직
ADMIN : 관리자용 HTTP 요청처리, 뷰 렌더링, 부트 실행

추후의 의존성을 위해서 고려하고 적용시켜야함
코어에 db관련 의존성을 둘수도있고 api단에 둘수도있고..

모든 프로젝트들이 동일한 버전을 가져가게끔 대부분 만들고있음

관리가 용이해지는 시점이 분리의 시점이라고 생각
